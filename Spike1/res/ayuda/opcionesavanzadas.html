Introducción

XRayDetector permite configurar una serie de parámetros que permite ajustar la aplicación en función al tipo de análisis o entrenamiento deseado.
A continuación se especifican con detenimiento los diferentes parámetros y sus posibles opciones a elegir. [INCLUIR PANTALLAZO]

Tamaño de la ventana

XRayDetector permite establecer diferentes tamaños de ventana para realizar el proceso de análisis de la imagen así como para determinar el tamaño de la ventana utilizado para el proceso de entrenamiento.
Los posibles valores entre los que se permite elegir son:

  • 8 x 8
  • 12 x 12
  • 16 x 16
  • 24 x 24
  • 32 x 32
  • 48 x 48
[INCLUIR PANTALLAZO]

NOTA: Este parámetro influye en el tiempo que le llevará a la aplicación analizar una imagen o entrenar un clasificador. A mayor tamaño de ventana menor tiempo y viceversa.

Tamaño del salto

XRayDetector permite definir el tamaño del salto que efectuará la ventana entre ventana y ventana.
Este valor puede ser definido mediante una barra de desplazamiento que permite definir un valor entre un tamaño mínimo de un 10% de la ventana y un máximo del 100% de la ventana.

NOTA: Este parámetro influye en el tiempo que le llevará a la aplicación analizar una imagen o entrenar un clasificador. A mayor tamaño del salto menor tiempo y viceversa.

Tipo de detección

XRayDetector permite establecer que tipo de detección se va a llevar a cabo.
Las opciones a elegir son las siguientes:

  • Normal: Se realiza el cálculo de características de texturas sobre una ventana deslizante, que va recorriendo la imagen.
  • Normal + Umbrales locales: Se realiza el cálculo de texturas de forma normal, pero al terminar se calculan los umbrales locales y se marcan como defecto aquellas zonas que fueron marcadas como defecto en la detección normal y que, además, fueron marcadas como blancos en los umbrales locales.
  • Blancos en Umbrales locales: Se saca la imagen segmentada con los umbrales locales y se centra un número de ventanas sobre las zonas que fueron marcadas como blancos. En estas ventanas se calculan las características de texturas.
[INCLUIR PANTALLAZO]

Ventana de entrenamiento

Se puede determinar el tipo de entrenamiento a realizar mediante la selección del tipo de ventana que llevará a cabo este proceso.
Las opciones a elegir son las siguientes:
  • Ventana aleatoria: Una ventana extrae la información de un conjunto de pixeles en la imagen. La posición de esta ventana en la imagen es aleatorio. Este proceso se repite hasta que se genere un conjunto de datos de entrenamiento suficientemente grande para poder ser utilizado por la aplicación.
  • Ventana deslizante: Una ventana extrae la información de un conjunto de pixeles en la imagen. Este proceso a diferencia de la ventana aleatoria se realiza de forma secuencial, es decir, la ventana recorrerá la imagen de principio a fin de forma secuencial hasta completarla generando un conjunto de datos con los que entrenar un clasificador.

Tipo de clasificación

Cuando se realiza el proceso de entrenamiento y detección de defectos, puede determinarse el tipo de clasificación que la aplicación determinará a la hora de definir si una ventana ha sido marcada como defectuosa o no. Esto determina si el ARFF generado contendrá un parametro numérico o booleano para definir si se ha encontrado un defecto o no dentro de la ventana.
Las opciones a elegir son las siguientes:
  • Clases Nominales: Una ventana será clasificada como defectuosa o no mediante en indicador TRUE o FALSE.
  • Regresión: Una ventana será considerada o no como defectuosa según el número de píxeles detectados como defectuosos dentro de la misma. En este caso lo que se almacena serán el número de píxeles detectados como defectuosos.
[INCLUIR PANTALLAZO]

Heurística ventana defectuosa

XRayDetector permite elegir entre diferentes heurísticas a seguir para decidir si una ventana será clasificada como defectuosa o no.
Las opciones a elegir son las siguientes:
  • Porcentaje píxeles malos en la ventana: Si el número de píxeles defectuosos encontrados en la ventana supera un umbral mínimo definido por el usuario, la ventana entera será marcada como defectuosa.
  • Porcentaje vecinos malos píxel central: Determina qué porcentaje de vecinos con respecto al pixel central dela ventana pueden ser considerados como defectuosos. Este umbral puede ser definido por el usuario. Si se supera dicho umbral, la ventana entera será marcada como defectuosa.
[INCLUIR PANTALLAZO]

Porcentaje píxeles defectuosos

Permite definir el porcentaje de píxeles defectuosos utilizados como umbral. Esto permitira determinar si una ventana será clasificada o no como defectuosa durante el proceso de análisis de la imagen.
Este valor puede ser definido mediante una barra de desplazamiento que permite definir un valor entre un tamaño mínimo de un 10% de la ventana y un máximo del 100% de la ventana.
[INCLUIR PANTALLAZO]
